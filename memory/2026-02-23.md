# 2026-02-23

## Fix: `.env` output directory settings not respected

### Problem
`GmailIngestorSettings` uses `env_prefix="GMAIL_"` (pydantic-settings), which means all fields expect env vars prefixed with `GMAIL_`. However, `.env` defined `OUTPUT_MARKDOWN_DIR`, `OUTPUT_RAW_DIR`, `DATABASE_PATH`, and `LOG_LEVEL` without the prefix — so they were silently ignored and defaults were used instead.

### Changes
- **`.env`** — Renamed `OUTPUT_MARKDOWN_DIR` → `GMAIL_OUTPUT_MARKDOWN_DIR`, `OUTPUT_RAW_DIR` → `GMAIL_OUTPUT_RAW_DIR`, `DATABASE_PATH` → `GMAIL_DATABASE_PATH`, `LOG_LEVEL` → `GMAIL_LOG_LEVEL`
- **`.env.example`** — Applied the same renames for consistency
- **`README.md`** — Added the four missing env vars to the configuration table

### Verification
- Settings now correctly load values from `.env` (confirmed `output_markdown_dir` = `../../output/markdown` instead of default `output/markdown`)
- All 225 tests pass
