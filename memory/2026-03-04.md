# 2026-03-04

## Change: Markdown file naming simplified

- **Before**: `{date}_{slug}_{id}.md` (e.g., `2024-01-15_weekly-newsletter_18a3f2b0.md`)
- **After**: `{slug}_{id}.md` (e.g., `weekly-newsletter_18a3f2b0.md`)
- Date prefix removed since it's already in YAML front matter
- Updated: `writer.py`, `test_writer.py`, README, SKILL.md, ARCHITECTURE.md

## Change: historyId-based incremental sync for discovery

- **Before**: Every `discover` run re-scanned ALL message IDs via `messages.list`, relying on `INSERT OR IGNORE` to skip duplicates
- **After**: Uses Gmail `history.list` API with stored `historyId` to discover only new messages since last sync
- New `sync_state` table in SQLite stores per-label `historyId`
- New `HistoryExpiredError` exception for 404 from history.list
- New `GmailClient` methods: `get_profile_history_id()`, `discover_message_ids_incremental()`
- Refactored `run_discovery()` into `_discover_full()` + `_discover_from_history()`
- CLI: `--full-sync` flag on `fetch` and `discover` commands
- Falls back to full discovery on: first run, expired historyId (404), `--query` flag, `--full-sync` flag
- historyId captured BEFORE discovery starts (so messages arriving during discovery are caught next time)
- Updated: `exceptions.py`, `tracker.py`, `gmail_client.py`, `ingestor.py`, `cli.py`, ARCHITECTURE.md, README.md, SKILL.md
- Tests: 21 new tests (280 total, all passing)
